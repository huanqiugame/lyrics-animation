动画引擎应该尽可能丰富应用CSS可以实现的功能，供用户进行丰富的调整和自定义。

# 歌词行的动画

歌词行的动画应该至少包含以下功能：

- 文字透明度变化；
- 文字模糊度程度变化；
- 文字颜色调整（包括纯色、渐变等）；
- 文字阴影变化；
- 文本框边框和背景，和他们的颜色、圆角等（参考CSS的`border`和`background`定义内容）；
- 文字的描边；
- 文字提前于或落后于歌词时间触发动画的功能（使用相对时间。例如相对于歌词文件中定义的时间提前100ms显示；延迟1500ms期间逐渐变模糊透明）；
- 文字大小；
- 字体粗细；
- 字体；
- 文本对齐方式（上中下、左中右）；
- 锚点位置，例如可以设置为文本框中心、四角、四条边的中点、画布的四角等；
- 关于XYZ轴的变换（如移动、3D旋转，覆盖关系等），原点即为上述锚点位置。

以上只是一部分，肯定还会有我没有想到的。为了便于日后修改和新增，请你尽可能模块化以上功能。

每一个功能，只要是随时间变化的效果，都应该可以定义曲线。至少应该预设8种动画曲线。同样模块化定义。

## 动画定义

- 一个动画组定义。
    - 从
        - `(字开始时间|字结束时间|行开始时间|行结束时间)`
        - `(前|后)`
        - `<整数>`ms
    - 时开始，到
        - `(字开始时间|字结束时间|行开始时间|行结束时间)`
        - `(前|后)`
        - `<整数>`ms
    - 时结束，应用
        - 一个动画定义。
            - 一个动画类型（如不透明度、颜色）。
            - 针对不同动画类型显示不同的菜单。
            - 一个曲线。
        - 另几个动画定义，格式与上面这个动画定义相同。
- 另几个动画组，格式与上面这个动画组定义相同。

举例（一个动画组）：从 字开始时间 前 500ms 时开始，到 字开始时间 前 0ms 时结束，应用 不透明度 从 0% 到 100% 的动画，使用 ease-in 曲线；以及应用 Y轴 +10px （从下往上移动10px）的动画，使用 贝塞尔 cubic-bezier(0.25, 0.10, 0.25, 1.00) 曲线。

# 歌词字的动画

独字的动画功能特性应该和上面的整行动画一样，包括各种字体、文字、颜色、描边、模糊等等。如果定义了相关的独字动画，则独字动画应该覆盖整行动画。否则不应覆盖。例如如果独字单独增大了其字体大小，没有定义延后消失，则这个字应该和同一行的其它字一样，同时出现和消失，只是稍微变大了而已。注意此处“未定义”：不是说定义逐字动画时就自动拉取行的动画作为字的预设，而是在解析时，遇到字没有定义时自动回退到行的定义。应保证行动画在修改时，可以在不修改字的数据结构的情况下，字的未定义的样式依然能够和行保持一致，避免突兀。

# 动画预设

尽管动画预设应该可由用户自己定义，但是我们需要一些基础的动画和配置来保证首次可用性。

对于每个字，在字开始时间时出现，在行结束时间时消失。字体设为系统字体，字号为32px（2.5rem），画布为黑色，字体为白色，无阴影、无描边，后出现的歌词默认在前出现的歌词上层，其余效果都不设定。

注意这只是系统最基础的一套设定，只在没有自定义配置或Default配置时作为最终的回退设置。都最终了，允许硬编码。

对于配置文件，我们需要定义一套默认的行动画和字动画，在初次启动时加载。样式的加载优先级是 行内字动画>行内行动画>自定义全局配置字动画>自定义全局配置行动画>Default全局配置字动画>Default全局配置行动画>硬编码动画效果。
