# 时间轴功能设计

以下是针对时间轴功能的详细描述：

- 移除现在界面中的“时间轴”标题和它所在的一整行空间。
- 添加时间轴。时间轴包含一个播放头，贯穿整个时间轴区域，指示当前的播放位置。可拖动其顶部或点击其顶部的整行区域改变播放位置。
    - 定义一个调色盘，包含至少12种不同的颜色，按渐变顺序从0–11排列。
        - 至少应包含背景色（浅）、文字色（深）、强调色（主色）、次要色等。请按语义正确命名，我这几个只是示例的应用位置。
        - 如果现有框架已经有了对应的调色盘，补齐到12个颜色即可。
    - 时间轴底部一行是歌词内容的显示区域。
        - 普通歌词在这一行内铺开（铺开的歌词称为主要歌词），每一个词占据一块位置（称为词块）。
            - 词块应为半透明的，横向有边框。
            - 通常情况下词块不应有重叠部分。但如果TTML文件中出现重叠的时间部分，那时间轴中就重叠。
        - 词块上1/6部分覆盖着一条很细但很长的条带，横跨多个词块，用于指示多个词块属于一句歌词。
            - 它应该是黑色或其他不抢眼的颜色。
            - 这个条带应该有横向的边框，用于区分一句歌词的断开处。
        - 词块下1/6部分覆盖着一条很细的条带，用于指示背景歌词的出现位置。
            - 它应该是亮紫色或其他能够引人注目的颜色。但不要太明显。
            - 这个条带应该有横向的边框，用于区分背景歌词词块的断开处。
            - 在背景歌词不存在时条带显然也不应该存在。考虑“覆盖”这一特性：在条带不存在时它就应该不存在，不可以被点到。
            - 在悬浮时纵向稍稍变大，指示可以点击。
        - 上述的两个条带都应该是覆盖在词块上，而不是单独占用一行空间。
        - 在点按任一背景歌词条带时，整个界面的普通歌词显示和背景歌词显示互换，包括下文的其他内容都应该根据新的主要歌词互换。
        - 注意鼠标指针的样式和功能变化。
    - 歌词内容向上一行是动画显示区。应有两个视图。视图1：动画组视图。
        - 定义：动画组“类”是指在编辑器中定义的一个动画组。它会一次定义应用于一个或多个歌词行或字上。特别地，我们将定义于全局动画组部分的动画组类称为全局动画组类。
        - 定义：动画组“实例”是指真实应用在某个行或字的动画组。一个动画组类应用于几个歌词行或字就产生多少个动画组实例。
        - 包含任意“无穷”时间定义的动画组类（之前 或 之后 或 之前和之后）都被下述条件完全忽略而从不显示。
        - 动画组视图应当按优先级显示多个全局动画组“类”。每个动画组类占据单独的一行。最低优先级的全局动画组“类”位于最底部，最高优先级的全局动画组“类”位于最顶部，其他全局动画组“类”按优先级在二者之间依次排开。
            - 在一个全局动画组“类”内，如果动画组实例的持续时长出现重叠，则分开多行显示，时间上先出现的动画组更靠下。由于此处的重叠是事实上是时间上的重叠，没有动画组优先级问题，这样处理是合理的。
                - 例如，如果歌词A（00.00–01.00）歌词B（01.00–02.00），设置字全局动画组为字开始前0.5秒到字结束后0.5秒，则会在00.50–01.50处发生动画组实例重叠。此时，歌词A的动画组实例应显示在靠下的一行中，歌词B的动画组实例应显示在靠上的一行中。
            - 行的高度取决于重叠的动画组实例占据几行。每个动画组实例的高度是固定的。
        - 在动画组“类”的上方排列的是零散的、单独针对行或字定义的动画组（散列动画组实例）。
            - 大致要求和上述全局动画组类相同：更高的优先级应该在更靠上的行中，应用于不同歌词的散列动画组实例如果发生重叠，则向上寻找空间。
            - 不同之处在于，散列动画组实例的排列是零散的，只要不发生重叠，同一行内可能存在多个不同动画组类的实例；而在全局动画组类的实现中，同一行内只可能存在一个动画组类的实例。
            - 换句话说，我们在尊重同行或同字动画组优先级顺序、且不重叠的前提下，将所有散列动画组实例尽可能放在这一区域最靠下的位置，而不是无限向上堆叠。
            - 注意，这一部分的动画组实例不应该被放到 全局动画组类区域 中。
        - 每个动画组实例上写的是对应动画组类的名字，双击这个动画组实例应当可以在动画组编辑器中显示出来对应的动画组类。
        - 视图左侧是一个侧边栏，展示这一行对应的内容。从下到上依次是：全局动画组类名（例如动画组3、动画组2、动画组1）、散列动画组（显示为“其他动画组”）。
        - 视图横向和纵向空间不足时都可以滚动。
        - 视图应该支持横向和纵向缩放。缩放效果类似于剪辑软件里那样。
        - 染色方面，根据调色盘：
            - 全局动画组类从下往上依次染色，同种全局动画组类的所有实例都应用一种颜色，便于区分。
            - 散列动画组中，应用在同一个行或字上的动画组实例采用相同的颜色，不同的行或字动画组采用不同的颜色。
        - 在选中一个歌词行或字时，
            - 除应用于这个行或字的动画组实例仍然保持彩色，其他动画组实例都应变为灰色。
            - 时间轴背景方面，除了选中的歌词行或字持续的时间区间仍为黑色，其他部分都改为灰色。
            - 点击其他歌词行或字可切换选中，再次点击、点击空白处或点击任意动画组可取消选中。
    - 视图2：动画通道视图。
        - 在页面一角设置“按通道显示”菜单，允许多选任意通道。在有通道被选中时启用此视图，否则应用视图1。
        - 此视图按菜单中的选择顺序，从下往上排列选中的通道行。例如，从歌词行往上可能是：不透明度行、模糊行、Y轴位移行。
        - 每一行内显示的是对应通道的值-时间变化曲线。数值越大，曲线上的点应该越靠上。在一个词块被选中时，所有行显示选中歌词的在对应通道的变化曲线。此变化曲线应该是真实的显示结果。意味着它是在按动画组优先级分析后得出的最终动画结果。可能出现跳变。
            - 事实上，我们可以认为，一个动画组通道（包含开始、结束时间、曲线和定义的值）就是两个关键帧。
            - 变化曲线上应该通过小菱形图标显示定义的关键帧。
                - 鼠标悬浮在关键帧图标上时会变为手指型，可以点击使其略微放大高亮选中，并同时高亮 它所在动画通道的另一个关键帧 和它们之间的曲线。
                - 双击关键帧图标和在动画参数面板中显示对应的动画通道。
        - 在没有任何词块被选中时，各行应显示一个文字占位符“请选中歌词查看关键帧曲线”。
        - 染色方面，根据调色盘：
            - 从下往上依次对各通道行进行染色。注意上述高亮的表述是指应用对应通道行的调色盘中的高亮色。其他颜色搭配在保证可读性的前提下自行配置。
        - 视图横向和纵向空间不足时都可以滚动。
        - 视图应该支持横向和纵向缩放。缩放效果类似于剪辑软件里那样。
    - 注意：鼠标指针的样式和功能变化。
        - 在空白处变为手掌型，可进行时间轴的拖拽移动。当然也应该支持滚轮和触控板的滚动/滑动操作。
        - 在歌词行、词块、动画组实例处变为手指型，指示可点击。
        - 其他方面按照标准规范进行设计。
